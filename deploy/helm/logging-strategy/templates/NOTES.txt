1. Get the application URL by running these commands:

{{- if .Values.ingress.enabled }}

  export INGRESS_HOST=$(kubectl get ingress {{ include "logging-strategy.fullname" . }} -o jsonpath='{.spec.rules[0].host}')
  echo "Open http://${INGRESS_HOST}/ (ensure DNS or /etc/hosts)"

{{- else if contains "NodePort" .Values.service.type }}

  From inside the cluster the Service works as usual. From your laptop (especially Minikube with the Docker driver on macOS), NodePort on $(minikube ip) is often not reachable; prefer one of:

    minikube service {{ include "logging-strategy.fullname" . }} -n {{ .Release.Namespace }} --url

    kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "logging-strategy.fullname" . }} 8080:{{ .Values.service.port }}

  Direct NodePort URL (works on some setups):

  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "logging-strategy.fullname" . }})
  export NODE_IP=$(kubectl get nodes -o jsonpath="{.items[0].status.addresses[?(@.type=='InternalIP')].address}")
  echo "http://${NODE_IP}:${NODE_PORT}"

{{- else if contains "ClusterIP" .Values.service.type }}

  kubectl --namespace {{ .Release.Namespace }} port-forward service/{{ include "logging-strategy.fullname" . }} 8080:{{ .Values.service.port }}
  echo "Then open http://127.0.0.1:8080"

{{- end }}

2. Uninstall: helm uninstall {{ .Release.Name }} -n {{ .Release.Namespace }}
